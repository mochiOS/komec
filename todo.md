### 1.1 関数・戻り値・return
- [ ] `fn f(...) -> T { ... }` の意味（最後の式を返すか / `return` 必須か）を決める
    -> `return` は任意で、最後の式が戻り値になる

- [ ] `return` の型チェック（`-> T` と一致するか）
    -> `return` の型は `-> T` と一致する必要がある

- [ ] `return` の到達性チェック（全パスで値が返るか）
    -> すべてのコードパスで値が返ることを要求する

- [ ] `void` 相当の扱いを決める
    -> `none`がvoid相当

### 1.2 可変長引数
- [ ] `xs: Int...` について
    -> `xs`は最後の引数で、0個以上の指定された型（int, stringなど）を受け取る

- [ ] `Int...` の要素型は将来 `Any...` を許すか
    -> 許さん。許すわけがなかろう。

- [ ] 可変長の受け取りを「配列として見る」構文（例: `xs.len`, `xs[i]`）をどうするか

### 1.3 制御構文（Swift っぽさに直結）
- [ ] `if` は式か文か（`let x = if ... { ... } else { ... }` を許すか）
    -> 式として使う。なぜならそっちのほうが簡潔にかけるから

- [ ] `while` / `for` の仕様
    -> `for i in a ~ b` は `i` が `a` から `b-1` までの整数を取る。 `while` は条件が真の間ループする。

- [ ] `match`（または `switch`）の仕様（enum / int / String など）
    -> `match`で整数、文字列、enumをパターンマッチできるようにする。将来的には構造体も。

## 6. bundle / recipe

- [ ] `bundle` の意味を明確化
    -> `bundle`は状態とそれに紐づく処理（レシピ）をまとめるための構造体のようなもの。状態が更新されるたびに関連するレシピが自動的に再評価される。

- [ ] `state`の型
    -> `state`は任意の型を持つことができる。型は明示的に指定するか、初期値から推論される。

- [ ] `recipe` の依存宣言と購読登録の仕様
    -> `recipe`は特定のstateに依存することができ、その状態が更新されるたびに自動的に再評価される。依存関係はコンパイル時に解析され、実行時には効率的な更新が行われるようにする。

- [ ] イベント/ループの責務分離
    -> イベントループは状態の更新とレシピの再評価を管理し、開発者は状態とレシピの定義に集中できるようにする。
